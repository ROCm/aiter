## 🔍 How to inspect installed hardware capability

Running the following command will give details on the availablity and number of compute units on your installed hardware.

```bash
rocminfo
```

## ℹ️ ROCm `rocminfo` Field Descriptions

The following sections describe each field generated by the `rocminfo` command.

---

### 🖥 Device Identification
- **Name** – Internal GPU architecture identifier (e.g., `gfx942`). Used by compilers and ROCm runtime to target the correct instruction set.  
- **Uuid** – Universally unique identifier for the GPU. Useful for distinguishing multiple GPUs in multi-GPU systems.  
- **Marketing Name** – Commercial product name, such as `AMD Instinct MI300X`.  
- **Vendor Name** – Vendor of the device (always `AMD` for ROCm-supported hardware).  

---

### ⚙ Capabilities and Profiles
- **Feature** – Lists supported GPU features like `KERNEL_DISPATCH`, which indicates the GPU can execute dispatched kernels.  
- **Profile** – The HSA profile supported. `BASE_PROFILE` indicates standard HSA/ROCm functionality.  
- **Float Round Mode** – Default floating-point rounding mode (e.g., `NEAR` = round-to-nearest). Important for consistent numerical behavior.  
- **Fast F16 Operation** – `TRUE` indicates hardware acceleration for FP16 (half-precision) operations.  

---

### 📋 Queue Information
- **Max Queue Number** – Maximum number of hardware queues that can run concurrently, affecting parallelism.  
- **Queue Min Size** – Minimum size (in bytes) for a command queue.  
- **Queue Max Size** – Maximum size (in bytes) for a command queue.  
- **Queue Type** – Type of queue supported (`MULTI` allows multiple simultaneous queues).  

---

### 🧩 Hardware Topology
- **Node** – Internal HSA node identifier for this device.  
- **Device Type** – Type of device (`GPU`).  
- **BDFID** – PCI Bus:Device:Function identifier (encoded). Useful for locating the GPU in PCIe topology.  
- **Internal Node ID** – ROCm’s internal identifier for the GPU.  

---

### 🗄 Cache Information
- **L1, L2, L3** – Sizes of the L1, L2, and L3 caches (in KB). Larger caches can improve performance by reducing memory latency.  
- **Cacheline Size** – Size (in bytes) of cache lines. Impacts memory coalescing and access efficiency.  

---

### 🚀 Compute Configuration
- **Chip ID** – Unique identifier for the GPU ASIC.  
- **ASIC Revision** – Revision number of the silicon.  
- **Max Clock Freq. (MHz)** – Maximum GPU clock frequency.  
- **Compute Unit** – Number of Compute Units (CUs). More CUs generally mean higher compute capacity.  
- **SIMDs per CU** – Number of SIMD units per CU. SIMDs execute wavefronts in parallel.  
- **Shader Engines** – Number of shader engine groups.  
- **Shader Arrs. per Eng.** – Shader arrays per shader engine.  
- **Wavefront Size** – Number of threads in a wavefront (`64` on most AMD hardware). Critical for performance tuning.  
- **Max Waves Per CU** – Maximum wavefronts that can reside in a CU.  
- **Max Work-item Per CU** – Maximum work-items a CU can execute.  

---

### 🧮 Workgroup and Grid Limits
- **Workgroup Max Size** – Maximum number of threads in a single workgroup.  
- **Workgroup Max Size per Dimension (x, y, z)** – Per-axis maximum dimensions for a workgroup.  
- **Grid Max Size** – Maximum total size of a grid for kernel launches.  
- **Grid Max Size per Dimension (x, y, z)** – Per-axis maximum grid size for kernel launches.  
- **Max fbarriers/Workgrp** – Maximum number of barriers available for synchronization within a workgroup.  

---

### 🧠 Memory and Host Interaction
- **Coherent Host Access** – Indicates if CPU and GPU memory accesses are coherent (`FALSE` = explicit synchronization required).  
- **Memory Properties** – General properties of the GPU’s memory system (HBM details, etc.).  

---

### 🔧 Microcode and IOMMU
- **Packet Processor uCode** – Version of the GPU packet processor firmware. Helps in debugging firmware-related issues.  
- **SDMA engine uCode** – Version of the firmware for the SDMA (System DMA) engine.  
- **IOMMU Support** – Indicates level of IOMMU support for device memory mapping.  

---

## 🖥️ Query Device Attributes via HIP

You can programmatically query GPU properties using the HIP runtime API:
```cpp
#include <hip/hip_runtime.h>
#include <stdio.h>

int main() {
    int deviceCount;
    hipGetDeviceCount(&deviceCount);
    printf("Number of GPUs: %d\n", deviceCount);

    for (int i = 0; i < deviceCount; ++i) {
        hipDeviceProp_t prop;
        hipGetDeviceProperties(&prop, i);
        printf("GPU %d: %s\n", i, prop.name);
        printf("  Compute Units: %d\n", prop.multiProcessorCount);
        printf("  Max Threads per Block: %d\n", prop.maxThreadsPerBlock);
        printf("  Total Global Memory: %zu MB\n", prop.totalGlobalMem / (1024*1024));
        printf("  Shared Memory per Block: %zu KB\n", prop.sharedMemPerBlock / 1024);
        printf("  Registers per Block: %d\n", prop.regsPerBlock);
    }
    return 0;
}
```
